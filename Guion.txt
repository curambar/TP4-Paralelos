[0:00 - 0:45] Introducción y Concepto
Visual: Screencast de la página del repositorio en GitHub, transición rápida a IntelliJ mostrando la clase Buffer.java.
Audio: "En el desarrollo de Sistemas Distribuidos, coordinar tareas simultáneas es un desafío. Hoy resolvemos el problema del Productor y Consumidor. Modelaremos una App de Pedidos: los Clientes son productores que envían tickets, los Cocineros son consumidores, y el recurso compartido es una Bandeja central con capacidad limitada. Coordinamos el acceso de los hilos mediante Semáforos. Todo el código fuente está disponible en el repositorio de GitHub."

[0:45 - 1:45] Semáforos y Prevención de Deadlock
Visual: En el IDE, resaltar las definiciones de los semáforos. Luego bajar hasta producir() y sombrear el orden de los acquire().
Audio: "En la clase Buffer instanciamos tres semáforos. Dos de Sincronización: vacios inicia con la capacidad máxima de la bandeja y bloquea al cliente si no hay espacio físico; llenos inicia en cero y bloquea al cocinero si no hay tickets. El tercero es mutex, inicializado en 1, que funciona como llave y garantiza la exclusión mutua sobre la memoria.
El orden de los bloqueos es crítico. El cliente primero pide espacio con vacios.acquire() y luego toma la llave con mutex.acquire(). Si invirtiéramos este orden y la bandeja estuviera llena, el cliente se bloquearía reteniendo la llave en la mano. El cocinero no podría entrar a sacar una hamburguesa para liberar espacio, y ambos quedarían congelados para siempre. Mantener este orden estricto previene el Abrazo Mortal o Deadlock."

[1:45 - 2:15] Arquitectura Web
Visual: Resaltar el método toJson() y la variable ConcurrentHashMap en el IDE.
Audio: "Para la interfaz, aplicamos el principio de Separación de Responsabilidades. Java funciona como motor lógico y servidor HTTP nativo, empaquetando el estado actual en un archivo JSON. Usamos un ConcurrentHashMap para rastrear si un hilo trabaja o espera, evitando cuellos de botella. El Frontend realiza polling consultando este JSON cada 200 milisegundos para renderizar en tiempo real."

[2:15 - 3:00] Demostración y Conclusión
Visual: Cambiar al navegador en localhost:8080. Ajustar sliders a 15 clientes y 1 cocinero, mostrando la saturación.
Audio: "En la aplicación, un destello verde indica visualmente qué hilo tiene el mutex. Al estresar el sistema con 15 clientes y 1 cocinero, la bandeja se satura rápidamente. El semáforo de espacios llega a cero y el Sistema Operativo suspende a los clientes restantes, que se tiñen de gris y quedan esperando. El sistema se auto-regula de forma robusta.
Utilizar semáforos nos obliga a diseñar el flujo del recurso y no solo pensar en bloqueos de memoria. Exponer este control de bajo nivel como una API REST demuestra que puede integrarse perfectamente con interfaces web modernas. Tienen el código en GitHub para revisarlo a fondo."
